# JalBandhu (SagarSetu) - Comprehensive Documentation

## ğŸ“‹ Executive Summary

**JalBandhu** (formerly SagarSetu) is a cutting-edge ocean hazard reporting and maritime safety platform specifically designed for INCOIS (Indian National Centre for Ocean Information Services) and coastal communities across India. This comprehensive platform bridges the critical communication gap between coastal citizens, maritime authorities, and emergency responders through AI-powered hazard detection, real-time social media monitoring, and community-driven reporting systems.

### Mission Statement
"To create safer coastal communities and maritime environments by leveraging advanced technology, artificial intelligence, and community participation to detect, report, and respond to ocean hazards in real-time."

### Key Value Propositions
- **Democratized Maritime Safety**: Every coastal citizen becomes a maritime safety observer
- **AI-Powered Early Detection**: Advanced machine learning for rapid hazard identification
- **Real-time Social Intelligence**: Continuous monitoring of social media for emerging threats
- **INCOIS Integration**: Seamless connection with India's premier ocean information services
- **Multi-lingual Accessibility**: Support for diverse coastal communities across India

## ğŸ¯ Project Overview

### Target Audience
- **Primary**: Coastal communities, fishermen, maritime workers, port authorities
- **Secondary**: INCOIS officials, Coast Guard personnel, disaster management authorities
- **Tertiary**: Marine researchers, environmental agencies, tourism sector

### Geographic Scope
- **Bay of Bengal**: West Bengal, Odisha, Andhra Pradesh, Tamil Nadu, Puducherry
- **Arabian Sea**: Gujarat, Maharashtra, Goa, Karnataka, Kerala
- **Andaman & Nicobar Islands**: Strategic maritime monitoring
- **Lakshadweep**: Island community safety

### Core Problem Statement
Coastal communities often lack direct channels to report marine hazards to authorities, while traditional monitoring systems miss ground-level observations. Social media contains valuable early warning signals that remain untapped. The platform addresses these gaps through intelligent aggregation and analysis.

```sql
CREATE TABLE reports (
    -- Basic identification
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    report_id VARCHAR(50) UNIQUE NOT NULL,
    
    -- Ocean hazard classification
    hazard_type VARCHAR(50) CHECK (hazard_type IN (
        'tsunami_events', 'storm_surge', 'high_waves', 'swell_surges',
        'coastal_currents', 'coastal_erosion', 'marine_debris',
        'unusual_sea_behavior', 'coastal_infrastructure_damage'
    )),
    
    -- Enhanced location data
    location JSONB NOT NULL,
    coordinates POINT, -- PostGIS spatial data
    coastal_zone VARCHAR(100),
    water_body VARCHAR(100),
    distance_from_shore_km DECIMAL(8,2),
    
    -- Marine-specific parameters
    wave_height_estimated DECIMAL(5,2),
    wave_height_category VARCHAR(20),
    water_level_change VARCHAR(20),
    current_strength VARCHAR(20),
    sea_state INTEGER CHECK (sea_state BETWEEN 0 AND 9),
    tide_information VARCHAR(30),
    
    -- Safety and impact assessment
    severity_level INTEGER CHECK (severity_level BETWEEN 1 AND 5),
    safety_threat_level VARCHAR(20),
    people_at_risk_count INTEGER DEFAULT 0,
    infrastructure_impact VARCHAR(30),
    fishing_activity_affected BOOLEAN DEFAULT FALSE,
    shipping_lanes_affected BOOLEAN DEFAULT FALSE,
    
    -- AI and verification
    ai_analysis JSONB DEFAULT '{}'::jsonb,
    ai_confidence_score DECIMAL(3,2),
    verification_status VARCHAR(20),
    verified_by_user_id UUID,
    
    -- INCOIS integration
    incois_alert_id VARCHAR(100),
    incois_severity_score DECIMAL(3,2),
    early_warning_correlation BOOLEAN DEFAULT FALSE,
    
    -- Smart camera integration
    is_smart_camera_report BOOLEAN DEFAULT FALSE,
    smart_camera_id VARCHAR(100),
    iot_sensor_data JSONB DEFAULT '{}'::jsonb,
    
    -- Timestamps and metadata
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    reported_incident_time TIMESTAMPTZ
);
```
